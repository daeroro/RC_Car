SEC 6 is as Freescale's Cryptographic Accelerator and Assurance Module (CAAM).
Currently Freescale powerpc chip C29X is embedded with SEC 6.
SEC 6 device tree binding include:
   -SEC 6 Node
   -Job Ring Node
   -Full Example
SEC 6은 프리 스케일의 암호화 가속기 및 보증 모듈 (CAAM)입니다.
현재 Freescale의 powerpc 칩 C29X는 SEC 6에 내장되어있다.
SEC 6 장치 트리 바인딩에는 다음이 포함됩니다.
    -SEC 6 Node
   -Job Ring Node
   -Full Example
=====================================================================
SEC 6 Node

Description

    Node defines the base address of the SEC 6 block.
    This block specifies the address range of all global
    configuration registers for the SEC 6 block.
    For example, In C293, we could see three SEC 6 node.
노드는 SEC 6 블록의 기본 주소를 정의합니다.
     이 블록은 SEC 6 블록에 대한 모든 전역 구성 레지스터의 주소 범위를 지정합니다.
     예를 들어, C293에서는 세 개의 SEC 6 노드를 볼 수있었습니다.

PROPERTIES

   - compatible
      Usage: required
      Value type: <string>
      Definition: Must include "fsl,sec-v6.0".
- compatible
       사용법 : 필수
       값 유형 : <문자열>
       정의 : "fsl, sec-v6.0"을 포함해야합니다.

   - fsl,sec-era
      Usage: optional
      Value type: <u32>
      Definition: A standard property. Define the 'ERA' of the SEC
          device.
    - fsl, sec-era
       사용법 : 선택 사항
       값 유형 : <u32>
       정의 : 표준 속성. SEC 장치의 'ERA'을 정의하십시오.

   - #address-cells
       Usage: required
       Value type: <u32>
       Definition: A standard property.  Defines the number of cells
           for representing physical addresses in child nodes.
  - # address-cells
        사용법 : 필수
        값 유형 : <u32>
        정의 : 표준 속성. 자식 노드의 물리적 주소를 나타내는 셀의 수를 정의합니다.

   - #size-cells
       Usage: required
       Value type: <u32>
       Definition: A standard property.  Defines the number of cells
           for representing the size of physical addresses in
           child nodes.
- #size-cells
        사용법 : 필수
        값 유형 : <u32>
        정의 : 표준 속성. 자식 노드의 물리적 주소 크기를 나타내는 셀 수를 정의합니다.

   - reg
      Usage: required
      Value type: <prop-encoded-array>
      Definition: A standard property.  Specifies the physical
          address and length of the SEC 6 configuration registers.
- reg
       사용법 : 필수
       값 유형 : <prop-encoded-array>
       정의 : 표준 속성. SEC 6 구성 레지스터의 실제 주소와 길이를 지정합니다.

   - ranges
       Usage: required
       Value type: <prop-encoded-array>
       Definition: A standard property.  Specifies the physical address
           range of the SEC 6.0 register space (-SNVS not included).  A
           triplet that includes the child address, parent address, &
           length.
- ranges
        사용법 : 필수
        값 유형 : <prop-encoded-array>
        정의 : 표준 속성. SEC 6.0 레지스터 공간의 물리적 주소 범위를 지정합니다 (-SNVS는 포함되지 않음). 하위 주소, 상위 주소 및 길이를 포함하는 세 쌍.

   Note: All other standard properties (see the ePAPR) are allowed
   but are optional.
    참고 : 다른 모든 표준 속성 (ePAPR 참조)은 허용되지만 선택 사항입니다.

EXAMPLE
	crypto@a0000 {
		compatible = "fsl,sec-v6.0";
		fsl,sec-era = <6>;
		#address-cells = <1>;
		#size-cells = <1>;
		reg = <0xa0000 0x20000>;
		ranges = <0 0xa0000 0x20000>;
	};

Job Ring (JR) Node

    Child of the crypto node defines data processing interface to SEC 6
    across the peripheral bus for purposes of processing
    cryptographic descriptors. The specified address
    range can be made visible to one (or more) cores.
    The interrupt defined for this node is controlled within
    the address range of this node.
 Child of the crypto node은 cryptographic descriptors 를 처리하기 위해 주변 버스를 통해 SEC 6에 대한 데이터 처리 인터페이스를 정의합니다. 지정된 주소 범위는 하나 이상의 (또는 그 이상의) 코어에서 볼 수 있습니다. 이 노드에 대해 정의 된 인터럽트는이 노드의 주소 범위 내에서 제어됩니다.

  - compatible
      Usage: required
      Value type: <string>
      Definition: Must include "fsl,sec-v6.0-job-ring".

- compatible
       사용법 : 필수
       값 유형 : <문자열>
       정의 : "fsl, sec-v6.0-job-ring"을 포함해야합니다.

  - reg
      Usage: required
      Value type: <prop-encoded-array>
      Definition: Specifies a two JR parameters:  an offset from
           the parent physical address and the length the JR registers.
  - reg
       사용법 : 필수
       값 유형 : <prop-encoded-array>
       정의 : 두 개의 JR 매개 변수를 지정합니다 : 부모의 실제 주소로부터의 오프셋과 JR이 등록하는 길이.

   - interrupts
      Usage: required
      Value type: <prop_encoded-array>
      Definition:  Specifies the interrupts generated by this
           device.  The value of the interrupts property
           consists of one interrupt specifier. The format
           of the specifier is defined by the binding document
           describing the node's interrupt parent.
  -  interrupts
       사용법 : 필수
       값 유형 : <prop_encoded-array>
       정의 :이 장치에서 생성 된 인터럽트를 지정합니다. interrupts 속성의 값은 하나의 인터럽트 지정자로 구성됩니다. 
        지정자의 형식은 노드의 인터럽트 상위를 설명하는 바인딩 문서에 의해 정의됩니다.

EXAMPLE
	jr@1000 {
		compatible = "fsl,sec-v6.0-job-ring";
		reg = <0x1000 0x1000>;
		interrupts = <49 2 0 0>;
	};

===================================================================
Full Example

Since some chips may contain more than one SEC, the dtsi contains
only the node contents, not the node itself.  A chip using the SEC
should include the dtsi inside each SEC node. 
일부 칩에는 둘 이상의 SEC가 포함될 수 있으므로 dtsi에는 노드 내용 만 포함되며
 노드 자체는 포함하지 않습니다. SEC를 사용하는 칩은 각 SEC 노드 내에 dtsi를 포함해야합니다. 

 Example:

In qoriq-sec6.0.dtsi:

	compatible = "fsl,sec-v6.0";
	fsl,sec-era = <6>;
	#address-cells = <1>;
	#size-cells = <1>;

	jr@1000 {
		compatible = "fsl,sec-v6.0-job-ring",
			     "fsl,sec-v5.2-job-ring",
			     "fsl,sec-v5.0-job-ring",
			     "fsl,sec-v4.4-job-ring",
			     "fsl,sec-v4.0-job-ring";
		reg	   = <0x1000 0x1000>;
	};

	jr@2000 {
		compatible = "fsl,sec-v6.0-job-ring",
			     "fsl,sec-v5.2-job-ring",
			     "fsl,sec-v5.0-job-ring",
			     "fsl,sec-v4.4-job-ring",
			     "fsl,sec-v4.0-job-ring";
		reg	   = <0x2000 0x1000>;
	};

In the C293 device tree, we add the include of public property:
C293 장치 트리에서 공용 속성 인 include를 추가합니다.

	crypto@a0000 {
		/include/ "qoriq-sec6.0.dtsi"
	}

	crypto@a0000 {
		reg = <0xa0000 0x20000>;
		ranges = <0 0xa0000 0x20000>;

		jr@1000 {
			interrupts = <49 2 0 0>;
		};

		jr@2000 {
			interrupts = <50 2 0 0>;
		};
	};
