7. 능력단위요소 - 하드웨어 기능 분석하기

   RAM 이란 Random Access Memory 를 의미한다.
   이와 같은 구조 때문에 Linux Device Driver 에서도
   char 디바이스 드라이버와 block 디바이스 드라이버가 분류 된다.
   RAM 은 block 디바이스 드라이버로 분류되는데 그 이유를 기술하시오.

char 디바이스 드라이버는 순서가 중요하기 때문에 주로 키보드 같은 것들에 해당한다.
RAM 은 이름 Random 에서도 알 수 있듯이 순서가 중요하지 않다.
그러나 정해진 Block Size(4KB) 를 최소 단위로 사용하기 때문에 block 디바이스 드라이버라고 한다.
여기서 4KB 의 단위는 물리 메모리의 최소 단위인 Page Frame 에 해당한다.

8. 능력단위요소 - 하드웨어 기능 분석하기

   쿼드 콥터 개발에 가장 빈번하게 많이 사용되는 센서 중에는
   I2C, SPI, CAN 등을 지원하는 센서들이 많이 들어간다.
   이 중에서 I2C 의 write 방식에 대해 기술해보도록 한다.

처음 START 비트를 보내고 Slave 가 될 장치의 주소를 보내고 Write 비트를 적어 보낸다.
Slave 로부터 ACK 응답을 받으면 Slave 장치 내부에 있는 레지스터를 보내고 ACK 응답을 기다린다.
응답을 받으면 레지스터에 쓸 값을 적어서 보내고 ACK 응답을 기다린다.
ACK 응답을 Slave 로부터 수신하면 STOP 비트를 보내서 Write 를 중단한다.

9. 능력단위요소 - 하드웨어 기능 분석하기

   아래 회로는 Buck Converter 회로에 해당한다.
   이 회로에서 인덕터와 다이오드의 기능에 대해 기술하도록 한다.

스위치가 ON 되어 있는 상태에서 다이오드는 역 바이어스 되어 전류의 흐름을 일정한 방향으로 고정시켜준다.
스위치가 OFF 되면 인덕터는 역기전력에 의해 음의 전압을 가지지만
다이오드의 0.6V 로 인해 전위차가 존재하여 ON 에서의 전류 흐름과 같은 방향을 가지게 된다.

    앞서 살펴봤던 Buck Converter 회로에 대한 문제다.
    기본적인 RL 회로의 미분 방정식은 아래와 같이 나타난다.
    그러나 우리의 해석에서는 리플 전압 및 전류를 구하기 위해 아래와 같은 식을 사용한다.
    어째서 이와 같은 식을 사용하게 되는 것인가 ?

DC-DC 컨버터들은 스위치 On, Off 를 굉장히 빠르게 수행하게 된다.
그렇다면 아주 짧은 시간이라고 볼 수 있고 이 짧은 시간을 근사하는데 맥클로닌 급수를 사용할 수 있을 것이다.
즉 RL 회로에서 도출 된 식에 맥클로닌 급수를 적용한 식을 우리가 최종적으로 사용하고 있을 뿐이다.
