1. TI 사의 제품을 개발하기 위해 만드시 필요한 SW 는 무엇인가 ?

Code Composer Studio(CCS) 와 각종 다운로드 케이블 드라이버들

2. FPGA Zybo 에 Baremetal 프로그래밍을 수행하기 위해 필요한 SW 는 무엇인가 ?

Vivado, Vivado SDK, 그 외 FPGA 용 케이블 드라이버들

3. 새로운 전원 회로를 설계하기 위해 필요한 SW 는 무엇인가 ?

PSIM 과 OrCAD, 그리고 PCB 전용 툴(Eagle, PADS, Altium Designer 등등)

4. 리눅스 커널을 컴파일 하기 위해 make menuconfig 를 수행 한 이후
   make 를 눌러서 커널 빌드를 수행하고 있다.
   ncurses 와 관련된 에러가 발생했다면 어떻게 해야 할까 ?

sudo apt-get update 명령어를 수행하고
이후에 sudo apt-get install 'ncurses 관련 라이브러리들' 을 통해서 라이브러리를 설치한다.
추가적으로 요구 도는 라이브러리가 존재한다면 역시 설치한다.

5. 프로세스 1 개로 Pipe 와 read(), write() 를 수행하는 경우를 생각해보자.
   이 경우 발생 할 수 있는 문제점들을 생각해보자!

기본적으로 read() 는 Blocking 함수에 해당하므로
한쪽의 read() 연산이 완료되기 전까지 다른 작업을 수행 할 수 없다는 문제점이 존재한다.

6. 앞선 Pipe 문제에서 발생했던 문제를 해결하기 위해 어떠한 작업이 필요한지 기술하라.

fcntl() 시스템 콜을 통해서 해당 파일 디스크립터의 작업을 전부 Non Blocking 작업으로 바꾼다.
그러면 read() 연산을 할 것이 없다면 자동으로 다른 작업을 수행하도록 만들 것이다.

7. stat() 시스템 콜이 가지고 있는 문제점을 기술하고 해결책을 제시하라.

stat() 시스템 콜은 링크 파일에 대한 정보를 제대로 가져오지 못하는 문제점을 가지고 있다.
이 문제점을 해결하기 위해서는 lstat() 시스템 콜을 사용해야 한다.

8. 임베디드 애플리케이션을 개발 할 때 제일 신경써야 하는 것은 무엇일까 ?

작업을 하다보면 점점 소스 코드의 분량이 늘어나게 된다.
그러면 컴파일 작업을 자동화시켜줄 툴이 필요하다.
즉 Makefile 과 같은 make 유틸리티를 잘 활용 할 수 있어야 한다.
이것을 나름 자동화 시켜주는 것이 CLion 이라는 Linux 상에서의 IDE 툴에 해당한다.
결론적으로 빌드 자동화 툴에 대해 신경을 써야 한다.

9. TI DSP 에서 UI 프로그램을 개발하는 경우 무엇을 사용하는 것이 좋을지 기술하도록 한다.

TI DSP 상에서 지원하는 것이 OpenCV, OpenCL, Qt5, OpenGLES 2.0 에 해당한다.
그러므로 UI 작업은 QT 를 가지고 진행하는 것이 좋다.

10. 우리의 프로젝트는 FPGA + DSP + MCU 로 구성된 복합 프로젝트다.
    이 상황에서 DSP 가 중앙 처리를 담당한다 가정하고 휴대폰으로 이를 제어한다고 가정하자.
    어떻게 해야 이것을 가능하게 할 수 있는지 기술해보자.

TI DSP 는 WL1837 이라는 고성능의 PCB 멀티 밴드 안테나를 제공한다.
안드로이드로 앱을 만들고 무선 통신으로 둘 사이의 데이터 경로를 뚫는다.
그리고 잡음에 견고하게 만들기 위해 FPGA 와 DSP, DSP 와 MCU 사이에 CAN 통신을 수행하여 명령을 전송하도록 한다.
