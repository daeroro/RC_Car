1. 능력단위요소: 임베디드 시스템 단위 테스트하기

   단위 테스트를 수행하는 이유가 무엇인가 ?

각각의 임베디드 단위 모듈이 정의되고 그 하나의 단위 모듈이 제품을 구동하기위한
최소 기능을 수행하는 소스코드로써 정확성 및 적합성을 검증하기 위해 수행한다.

2. 코드를 분리하고 파일을 분리하는 작업은 단위 테스트와 어떠한 관계가 있을까 ?

기능별로 함수가 분리되어 있고 파일이 분리되어 있다면
필요한 기능을 테스트하기 위한 main 함수(테스트 파일)를 만들어서
각각의 기능들이 잘 동작하는 구동 확인을 하기 수월 할 것이다.

3. 최종 완성된 제품을 변경하는 것보다 시제품이나 모델을 변경하는 것이 좋은 이유는 무엇인가 ?

임베디드 시스템에서 최종 제품이란 회로 설계를 통한 PCB 기판등의 각종 HW 와 여기에서 구동되는 SW 를 모두 포함한다.
그렇기 때문에 완성된 최종 제품을 변경하는 작업은 비용이 크다고 볼 수 있다.

4. 단위 테스트에서 정적 테스트란 무엇인지 기술하시오.

소프트웨어 실행 없이 소스코드, 설계 문서, 요구 사항 정의서를
수 작업(manual) 및 자동화(automatic) Tool 을 사용하여 Test 하는 방법이다.

5. 단위 테스트에서 동적 테스트란 무엇인지 기술하시오.

소프트웨어 프로그램이나 시스템 실행을 통하여 무엇이 어떻게 수행되고 동작되는지 Test 하는 방법이다.

6. 테스트 프로세스 절차에 대해 순서대로 열거하라.

요구 분석, 테스트 계획, 테스트 설계, 테스트 수행, 테스트 평가

7. 테스트 케이스에서 기대값이 무엇인지 기술하시오.

테스트 진행 후, 기대했던 동작을 수행했는지 판단하는 근거

8. 테스트를 평가하는 단계에서 결함 자체 평가에 대해 기술하시오.

테스트 수행 동안 발견된 결함 유형, 수, 발견율, 결함의 심각도 등을 분석하여 시스템의 품질 평가하는 과정이다.

9. 능력단위요소: 임베디드 시스템 통합 테스트하기

   시스템 통합은 HW 와 SW 를 모두 포함해서 수행하는데 통합 테스트의 정의에 대해 기술하시오.

개발된 단위 모듈을 결합해서 어떠한 방식으로 통합하여 하나의 완성된 프로그램
및 시스템으로 구성하기 위한 과정에 수행되는 테스트로서
모듈 간 인터페이스와 관련된 결함들을 발견하고 제거하면서 개발 모듈들을 통합시키는 테스트 활동이다.

10. 통합 테스트 중 상향식 통합 방식의 장점에 대해 기술하시오.

개발 초기에 통합이 가능하여 테스트가 용이하며,
인터페이스 문제의 조기 발견이 가능하기 때문에 개발 완료 후 발견된 것 보다 수정 비용이 적게 든다.

11. 시스템 테스트에서 학습성이라는 용어가 무엇인지 기술하시오.

사용자가 소프트웨어의 사용법을 배워 명시된 목적을 달성할 수 있는 정도

12. Xilinx Zynq Zybo 보드에서 주변 회로를 구성하고 드라이버가 잘 동작하는지 확인하고자 한다.
    이때 수행하는 일련의 절차를 기술해보도록 한다.

* Vivado SDK 를 킨다.
* 적절한 IP 를 설계한다.
* 설계 IP 의 Bit Stream 을 생성한다.
* Petalinux 를 활용하여 디바이스 드라이버(코드)와 이를 구동시킬 유저 코드를 작성한다.
* 리눅스 4.x 대 커널을 빌드하고 SD 카드에 복사한다.
* Zybo 보드에 SD 카드를 넣고 점퍼 설정을 한다.
* 보드에 전원을 인가하고 유저 코드를 실행하면 주변 회로의 구동을 확인 할 수 있다.

13. 능력단위요소: 시스템 테스트 지원하기

    임베디드 시스템에 대한 테스트를 수행할 때 모든 테스트 정보를 DB 에 저장하고 관리해야 한다.
    이 경우에 어떠한 내용들이 포함되어 있어야 하는가 ?

테스트 날자 및 시간, 테스트 수행 회수, 데이터 기록 회수, 테스트 계획 변경, 테스트 결과 보고서

14. 테스트를 수행 할 때 시뮬레이션을 수행하는 이유는 무엇인가 ?

설계한 내용에 대한 증명, 최적화 된 설계를 구성하기 위함이다.

15. 피드백 시뮬레이션의 예가 실제 어떤 부분에서 활용이 되는지 기술하시오.

모터의 속도를 제어하기 위한 PI 제어기,
조향을 제어하기 위한 PID 제어기,
항공기의 자세 제어를 위한 PD 및 PID 제어기,
DC-DC 컨버터의 Soft Start 를 위한 PID 제어기,
PLL 제어 등등

16. 프로토타이핑을 수행하는 이유를 기술하시오.

시뮬레이션 모델의 유효성을 검증, 시스템이 요구 사항을 만족하는지 검증하기 위함이다.

17. 능력단위요소: 임베디드 시스템 버그 수정하기

    전원 회로(DC-DC) 구현상에 문제가 발생했을 경우 어떤식으로 버그를 수정해야 하는가 ?

일반적으로 DC-DC 회로에서 발생 할 수 있을만한 문제는 LC 필터 문제,
RLC 공진 문제, ESR 기생 저항 문제, 인덕터 과포화 문제등등이 존재 할 수 있다.
이들 문제를 하나씩 확인하며 해결해나가야 한다.

18. Linux 에서 Segmentation Fault 가 발생했다.
    이 문제를 해결하기 위해 어떻게 해야 하는가 ?

일반적으로 Segmentation Fault 는 잘못된 메모리 참조를 수행할 경우 발생한다.
Page Fault 가 Kernel 권한이 없는 User 에서 발생하여 Segmentation Fault 가 나는 것이므로
포인터 연산을 수행하는 코드 부분에 문제가 있다고 볼 수 있다.
이 부분을 유추하여 찾아서 수정해줘야 한다.

19. 형상 관리 자동화 도구가 필요한 이유에 대해 기술하시오.

데이터의 일관성을 유지하고 프로젝트의 원활한 업무 수행과
생산성을 높이기 위해 자동화된 도구를 사용하는 것이 좋다.
