1. 능력단위요소: 요구 문서 검토하기

   github 에 보면 Markdown 이라는 것이 존재한다.
   이 Markdown 이라는 것은 github 에서 무엇을 하기 위해 존재하는 것인지 기술하시오.

github 의 README.md 에 Markdown 을 활용해서 해당 repo 에 대한 정보를 기록 할 수 있다.
다운로드부터 설치 활용하는 튜토리얼을 적을 수도 있고
내부 아키텍처에 대한 설명을 적을 수도 있다.
즉 문서화 작업을 위해 사용 된다.

2. 요구 사항을 분류하면 기능적 요구 사항과 비기능적 요구 사항으로 나뉜다.
   이 중 기능적 요구 사항에 대해 기술하시오.

이는 수행될 기능과 관련되어 입력과 출력 및 그들 사이의 처리 과정과
목표로 하는 제품의 구현을 위해 소프트웨어가 가져야 하는 기능적 속성에 대한 요구 사항들이다.

3. 비기능적 요구 사항에 대해 기술하시오.

제품의 품질 기준 등을 만족시키기 위해 소프트웨어가 가져야 하는 성능, 사용의 용이성, 안전성과 같은
행위적 특성과 시스템의 기능에 관련되지 않는 사항으로 성능(응답 시간, 처리량),
사용의 용이성, 신뢰도, 보안성, 운용상의 제약, 안전성 등에 해당한다.

4. 요구 사항을 추출하는 기법에 대해 기술하시오.

요구 사항을 추출하는 기법으로는 질문 조사 등을 통한 인터뷰 방법, 
시나리오 기반 작성 방법 등이 존재한다.

5. 개발 방법론에 존재하는 여러 가지 모델들의 종류를 기술하시오.

폭포수 모델, V 모델, UML, 애자일 소프트웨어 개발

6. 소프트웨어 인증에서 GS 인증에 대해 기술해보도록 한다.

소비자와 기업이 우수한 SW 제품을 보다 잘 믿고 쓸 수 있도록
일련의 시험 테스트 과정을 거쳐 일정한 수준의 품질을 갖춘 SW 제품에게 국가가 부여하는 인증

7. 전자 제품 인증에 있어서 매우 중요한 CE 마크에 대해 기술하시오.

제품이 안전, 건강, 환경 그리고 소비자 보호와 관련된 유럽 규격의 조건들을 준수한다는 의미
수출품일 경우에도 이는 매우 중요하다.

8. CE 마크와 동일하게 중요한 FCC 인증에 대해 기술하시오.

미국연방통신위원회는 전파 자원의 효율적 이용 관리 방안의 일환으로
주요 전기/전자 제품에서 발생하는 EMI를 규제
수출품일 경우에도 이는 매우 중요하다.

9. 국내 KCC 인증에 대해 기술하시오.

13개(공산품, 전기, 정보 통신 등)의 법정 강제 인증 마크를 국가 통합 인증 마크 하나로 통합

10. 요구 사항 관리 도구중 하나인 Trello 를 github 및 slack 과 함께 활용하는 편이다.
    실제 프로젝트에서 이들을 구체적으로 어떻게 활용하는지 기술하시오.

Trello 는 Plan Board 같은 것에 할 일들을 쭉 열거 할 수 있기 때문에 해야 할 작업들이 한 눈에 보인다.
즉 전체 개발 과정에서 요구 사항에 관한 관리를 수행하기에 적합하며
github 은 형상 관리를 수행하므로 요구 사항의 구현 결과물을 git 에 커밋한다.
또한 slack 을 함께 활용하여 카카오톡 같은 것이 잘 지원되지 않는 PC 리눅스에서도
이를 적절하게 활용 하여 전반적인 프로젝트를 관리 할 수 있다.

11. 오픈 소스 SW 문서 관리 시스템 중 Plone 의 특징을 기술하시오.

CMS 컨셉의 플랫폼이고 메모리 효율이 아주 좋아서 아주 큰 파일 사이즈도 다룰 수 있다.
매우 많은 개발자가 지원을 하고 있는 믿을만한 플랫폼이며 엄청 빠르다.

12. 임베디드 시스템 아키텍처에 대해 기술하시오.

임베디드 기기의 추상적 개념으로 소프트웨어 소스코드나 하드웨어 회로 디자인 같은
상세한 구현 정보를 보여주지 않는 시스템의 일반화를 의미한다.

13. 임베디드 소프트웨어 아키텍처란 무엇인지 기술하시오.

소프트웨어 아키텍처를 통해 시스템 전체에 대한 가시성이 제공되고
시스템의 목적, 범위, 기능들을 명확히 할 수 있게 된다.

14. 알고리즘을 표현하는 방식들을 열거하라.

자연어 기술 방법, 순서도 표현 방법, 의사 코드 기술 방법, 프로그래밍 언어 표현 방법

15. 제품 설명서의 사용자 요구 사항에 관련된 요소를 모두 열거하시오.

정확성, 실용성, 검색성, 해결법, 시각화, 적용성, 경제성
