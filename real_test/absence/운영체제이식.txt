1. Xilinx FPGA 에 Linux 를 올리는 가장 쉬운 방법은 무엇인지 기술하시오.

petalinux 툴을 활용하여 프로젝트를 생성하고 custom app 을 만들고 패키징하여 SD 카드에 굽는 방식이다.

2. TI DSP 에 Linux Kernel 을 올리는 가장 쉬운 방법을 기술하시오.

SDK 와 같은 Toolchain 을 활용하여 README 를 확인하고
나와있는대로 Kernel 을 컴파일 한 이후에 SD 카드 생성 명령을 통해 굽고 DSP 에 장착한다.

3. TI MCU 에 FreeRTOS 를 올리는 가장 쉬운 방법을 기술하시오.

할코젠(HalCoGen)을 키고 사용하는 IC 에 적절한 옵션을 선택하고
FreeRTOS 옵션을 선택한 이후에 필요한 것들을 선정하고 프로그램을 생성하면 FreeRTOS 를 올릴 수 있다.

4. FPGA 에 전용 드라이버를 만들고자 한다.
   어떻게 해야 하는가 ?

우선 Vivado 를 키고 HDL 베이스로 HW 를 설계한다.
이후에 Kernel 에서 Device Tree 를 작성하고 Custom App 을 만들어서 이미지를 SD 카드에 패키징하면 된다.

5. DSP 에 특수한 Device Driver 를 올리고자 한다.
   어떻게 해야하는가 ?

SDK 를 풀고 해당 위치에 있는 리눅스 커널 소스로 간다.
개조하고자 하는 드라이버를 선정하고 적절하게 소스코드를 수정한 이후에 SD 카드를 굽는다.

6. TI DSP 가 홀 펀칭을 수행하게 만들어주려면 어디 부분의 옵션을 활성화 시켜줘야 하는가 ?

kernel config 에서 NAT 와 IP MASQUERADE 를 활성화시켜줘야 한다.

7. 자신이 FPGA 베이스로 설계한 새로운 Chip 에 FreeRTOS 를 올리고자 한다.
   반드시 포팅해줘야 하는 파트가 어느 부분들인가 ?

HW 의존성이 있는 코드들로 특정 장치를 활성화 한다거나
혹은 해당 HW 의 어셈블리 루틴이 있는 부분을 FPGA 로 설계한 CPU 에 맞는 명령어 포맷으로 맞춰야 한다.

8. 포팅을 할 때 전혀 손댈 필요가 없는 영역들은 어디인가 ?

HW 의존성이 없는 SW 파트들은 별도로 코드를 수정할 필요가 없다.

9. 운영체제 부팅시 초반부가 전부 어셈블리어로 구현된 이유가 무엇인가 ?

초반에 메모리 레이아웃이 잡혀있지 않기 때문에 어셈블리어로 기본적인 설정들을 수행해줘야 한다.

10. petalinux 의 기존 설정을 싹 다 날리고 처음부터 모든 설정을 다 하고자 한다.
    어떤 명령어를 입력해야 하는가 ?

petalinux -x distclean
